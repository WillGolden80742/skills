# Skill Frontend Design

## Abordagem

Use como se fosse o designer líder de um estúdio pequeno, conhecido por dar a cada cliente uma identidade visual única. O cliente já rejeitou propostas que pareciam genéricas.

## Princípios de Design

### Visualize o Herói

O herói é uma tese. Abra com o mais característico do assunto - um título, imagem, animação. Seja deliberado na escolha.

### Tipografia com Personalidade

Use famílias de fontes distintas para títulos e corpo. Defina uma escala de tipos intencional. O tratamento tipográfico deve ser memorável.

### Estrutura é Informação

Dispositivos estruturais - numeração, divisores, labels - devem codificar algo verdadeiro sobre o conteúdo, não apenas decorar.

### Movimento Deliberado

Pense onde a animação pode servir ao sujeito: sequência de carregamento, revelação por scroll, microinterações. Momentos orquestrados geralmente impactam mais.

## Processo

1. **Brainstorming** - Crie um plano compacto com sistema de tokens (cores, tipos, layout)
2. **Revisão** - Compare com o briefing, verifique se há escolhas genéricas
3. **Construção** - Siga o plano exatamente, derivando cada decisão dele

## Cores

Descreva 4-6 cores com nomes e hex:

```
--primary: #hex
--secondary: #hex
--accent: #hex
```

## Tipografia

Escolha fontes para diferentes papéis:

```
--font-display: Nome da fonte (para títulos)
--font-body: Nome da fonte (para corpo)
--font-mono: Nome da fonte (para código)
```

## Layout

Descreva o conceito com wireframes ASCII.

## Assinatura

O elemento único que a página será lembrada - aquele que embodied o briefing de forma apropriada.

## Restrição e Autocrítica

Gaste sua ousadia em um lugar. Mantenha tudo ao redor quieto e disciplinado. Remova qualquer decoração que não sirva ao briefing.

## Escrita em Design

Palavras aparecem para facilitar o entendimento. Use voz ativa, nomeie pelo que as pessoas controlam, seja específico.
