## Routing Algorithm

```javascript
function route(task, codebaseContext) {
  // 1. Extract keywords from task
  const keywords = extractKeywords(task);
  
  // 2. Check for exact skill matches
  for (const keyword of keywords) {
    if (SKILL_MAP[keyword]) {
      return { skill: SKILL_MAP[keyword], confidence: 'high' };
    }
  }
  
  // 3. Check for partial matches
  for (const [pattern, skill] of Object.entries(SKILL_MAP)) {
    if (task.toLowerCase().includes(pattern)) {
      return { skill, confidence: 'medium' };
    }
  }
  
  // 4. Use graphify to understand codebase and find relevant skills
  if (codebaseContext) {
    const relevantFiles = graphify.query(`Files related to: ${task}`);
    const projectType = detectProjectType(relevantFiles);
    return routeBasedOnProjectType(task, projectType);
  }
  
  // 5. Default to graphify for general coding tasks
  return { skill: 'graphify', confidence: 'low' };
}
```

## Project Type Detection

The router detects project types from the codebase:

- **WordPress Theme/Plugin**: presence of `style.css` header, `functions.php`, `wp-content/themes/`
- **React App**: presence of `package.json` with react, `src/components/`, `jsx`/`tsx` files
- **Node.js**: presence of `package.json`, `node_modules/`, `npm` scripts
- **Python**: presence of `requirements.txt`, `setup.py`, `__init__.py`, `.py` files
- **PHP**: presence of `composer.json`, `.php` files, Laravel/CodeIgniter patterns
- **Generic**: when no specific pattern is detected

## Confidence Levels

- **high**: Exact keyword match (e.g., "pdf" → processar-arquivos-pdf)
- **medium**: Partial match (e.g., "create a pdf document" → processar-arquivos-pdf)
- **low**: Project-type inference (e.g., WordPress project → tema-marketplace-trokapay)
- **fallback**: No match → graphify for analysis
